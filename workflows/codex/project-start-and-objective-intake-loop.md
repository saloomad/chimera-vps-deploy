# Project Start And Objective Intake Loop

## Trigger

Use this workflow when:

- a new project is starting
- a new major objective appears
- the user asks what to work on next and the answer may create a real initiative

## Inputs

- user goal
- current constraints
- source-of-truth files
- likely owner
- likely consumer

## Steps

1. Run the starter stack.
2. Decide whether this is a new project, new task, or only a continuation.
3. Create the objective contract.
4. Decide whether council is needed.
5. Define the first safe slice.
6. Define done criteria and proof shape.
7. Update project/task truth if the initiative is real.
8. Create a file-backed `plan.md` if the work will be multi-pass.

## Expected Outputs

- clear objective
- project/task decision
- first safe slice
- plan path for the next pass
