# Project Finish And Delivery Loop

## Trigger

Use this workflow when:

- finishing meaningful work
- shipping or handing off
- deciding whether the objective is truly complete

## Inputs

- objective contract
- current proof
- changed files
- remaining risks

## Steps

1. Verify done criteria against proof.
2. Run any missing proof or test checks.
3. Decide whether GitHub publish or shared sync is needed.
4. Update PM and continuity surfaces.
5. Run the interaction monitor if friction or misses appeared.
6. Record `complete`, `iterate`, or `blocked`.

## Expected Outputs

- honest completion decision
- current continuity and PM truth
- follow-up path when still open
