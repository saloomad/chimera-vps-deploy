# Reusable Pattern Capture Workflow

## Trigger

Use this workflow when a repeated failure, repeated task, or high-risk process should become a reusable skill or workflow.

## Inputs

- current request
- recent task/action history
- existing skills/workflows/docs
- lessons from earlier failures

## Steps

1. Name the pattern clearly.
2. Check whether the pattern is already covered by an existing skill or workflow.
3. Decide whether the right artifact is:
   - doc
   - workflow
   - skill
   - or a combination
4. Write the smallest useful artifact.
5. Update any related skill or instruction file so the new artifact is actually discoverable.
6. Add or update a tracked task if follow-through is still needed.
7. Update continuity and lessons if the pattern matters next session.

## Expected Outputs

- a reusable artifact on disk
- any linked task/kanban/action updates
- a short explanation of why this became reusable
