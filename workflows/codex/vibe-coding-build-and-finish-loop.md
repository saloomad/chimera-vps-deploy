# Vibe Coding Build And Finish Loop

## Trigger

Use this workflow when we are:

- building something
- fixing something
- refactoring something
- testing something meaningful
- shipping or finishing a real work slice
- organizing a messy work item into a safer delivery path

## Goal

Keep AI-assisted software work beginner-friendly, safe, small-step, and durably tracked.

## Start Hook

1. Restate the goal in plain English.
2. Read the smallest relevant truth set:
   - `DELIVERY_JOURNAL.md`
   - `PROJECT_REGISTRY.md`
   - `TASK_REGISTRY.md`
   - `ACTION_LOG.md`
   - `KANBAN.md`
   - `CONTINUATION.md` when continuity matters
3. Decide whether the request needs:
   - `prompt-upgrade-engineer`
   - `project-operations-manager`
   - `github-manager`
   - `objective-orchestration-loop`
   - `critical-change-guard`
4. Define the smallest safe next step.

## Build Hook

1. Keep the change focused.
2. Explain terms simply.
3. Prefer one idea per branch or commit.
4. Decide the proof shape before claiming success:
   - local check
   - integration check
   - bounded live proof if relevant
5. If behavior changed, prepare to use `system-change-verifier`.

## Finish Hook

1. Verify the result honestly.
2. Update task/action/continuity truth if the work mattered.
3. Rebuild generated PM front-door files if their source truth changed.
4. Decide whether the result needs:
   - monitoring
   - a follow-up task
   - a captured lesson
5. Give the result in plain English first.

## Required Companion Skills

- `objective-orchestration-loop` for non-trivial work
- `system-change-verifier` before claiming changed behavior is real
- `codex-continuity-enforcer` after meaningful work

## Common Optional Companion Skills

- `prompt-upgrade-engineer`
- `project-operations-manager`
- `github-manager`
- `critical-change-guard`
- `codex-task-and-project-capture`
- `codex-lesson-harvester`
- `monitoring-opportunity-detector`
- `response-structure-enforcer`

## Honest Enforcement Rule

This workflow is a strong instruction and workflow layer for cooperating agents.
Do not describe it as a universal runtime-native hook unless that proof exists.
