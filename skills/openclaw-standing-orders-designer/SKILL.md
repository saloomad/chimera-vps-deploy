# OpenClaw Standing Orders Designer

Use this skill when the system needs permission and boundaries for recurring work, not just a timer.

## What Standing Orders Are

- permanent operating authority for defined programs
- the rule layer above repeated tasks
- best paired with cron or another wake-up surface

## When To Use It

- recurring reports
- recurring audits
- recurring trading or monitoring authority
- escalation boundaries that should survive sessions

## Current Chimera Truth

- standing-order style rules mainly live in `AGENTS.md`, bootstrap notes, and shared skills
- some live OpenClaw instruction surfaces still need freshness cleanup to fully match current VPS truth

## Output Contract

Leave behind:

- program scope
- triggers
- authority boundaries
- escalation rules
- wake-up mechanism
- stop or approval boundaries
